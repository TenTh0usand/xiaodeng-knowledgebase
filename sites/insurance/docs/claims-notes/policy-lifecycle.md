## End-To-End Flow

```text
销售/代理人讲解
  -> 客户投保
  -> 信息录入系统
  -> 核保
  -> 承保
  -> 出具电子合同或纸质合同
  -> 客户签收回执
  -> 回执确认
  -> 保单进入保全服务
  -> 出险后进入理赔
```

## Key Roles

| Role | 中文 | Meaning In The Flow |
|---|---|---|
| Agent | 代理人 | 负责销售讲解、协助投保、客户服务。 |
| Applicant / Policyholder | 投保人 | 提交投保申请并缴纳保费，通常拥有保单权益。 |
| Insured | 被保险人 | 被保障的人，健康告知和理赔事故围绕其展开。 |
| Beneficiary | 受益人 | 出险后有权领取保险金的人，寿险中特别重要。 |
| Underwriter | 核保人员 | 判断风险能否承保、是否加费、除外或拒保。 |
| Claims Adjuster / Claims Examiner | 理赔审核人员 | 判断事故是否属于保险责任并完成理算。 |

## Policy Elements To Check

| Element | 中文 | Why Claims Cares |
|---|---|---|
| Policy number | 保单号 | 系统定位案件和合同。 |
| Effective date | 生效日 | 判断事故是否发生在保障开始后。 |
| Signature date / receipt date | 签收日/回执日 | 影响犹豫期起算，也能确认客户已收到合同。 |
| Policy term | 保险期间 | 判断事故是否在保障期内。 |
| Premium payment term | 缴费期间 | 判断是否欠费、宽限期、失效。 |
| Waiting period | 等待期 | 疾病类理赔要确认是否已经过等待期。 |
| Grace period | 宽限期 | 未交费但仍可能承担责任的期间。 |
| Sum insured | 保额 | 给付型责任和部分限额计算基础。 |
| Premium | 保费 | 与承保、续期、现金价值、扣费相关。 |
| Cash value | 现金价值 | 退保、保单贷款、减保、自动垫交时使用。 |
| Riders | 附加险 | 理赔时要看主险和附加险分别有哪些责任。 |
| Exclusions | 责任免除 | 决定不赔或部分不赔的重要依据。 |

## Dates That Are Easy To Confuse

| Date | 中文 | Claims Meaning |
|---|---|---|
| Application date | 投保申请日 | 客户提交投保信息的日期，不一定等于生效日。 |
| Underwriting approval date | 核保通过日 | 风险审核完成日期。 |
| Issue date | 承保/签发日 | 公司出具保单的日期。 |
| Effective date | 生效日 | 合同保障开始日期。 |
| Receipt date | 回执签收日 | 客户确认收到合同的日期。 |
| Free-look start | 犹豫期起算 | 通常与签收回执相关，具体看条款和监管规则。 |
| Waiting period end | 等待期结束日 | 疾病类保险责任真正开始稳定承担的重要节点。 |
| Accident date | 出险日 | 理赔判断的时间锚点。 |

## Underwriting To Policy Issue

核保的结果可能有：

| Result | 中文 | Meaning |
|---|---|---|
| Standard acceptance | 标准体承保 | 正常费率、正常责任。 |
| Extra premium | 加费承保 | 风险更高，但可以通过增加保费承保。 |
| Exclusion | 除外承保 | 某些疾病、部位或责任不保。 |
| Postponement | 延期 | 等待病情稳定、检查结果明确或时间经过后再评估。 |
| Decline | 拒保 | 风险超过公司承保范围。 |

理赔审核时遇到过往病史或健康告知问题，要回看投保时是否如实告知、核保结论是否有除外或特别约定。

## Receipt, Free-Look And Waiting Period

扫描笔记里“电子回执/纸质回执、保单签收日、生效日、犹豫期、等待期”是一组很重要的时间概念：

- **回执**：证明客户收到并确认合同。电子合同和纸质合同都可能有回执。
- **签收日**：常用于计算犹豫期。
- **生效日**：判断保险责任是否开始。
- **犹豫期**：客户可以在规定期间内撤销合同，通常与新单回访和客户权益保护有关。
- **等待期**：防止带病投保，疾病类理赔尤其要查。

## Policy Servicing After Receipt

回执 OK 后，保单通常进入保全服务阶段。常见保全项目：

| Service | 中文 | Claims Connection |
|---|---|---|
| Beneficiary change | 受益人变更 | 身故理赔要看最新有效受益人。 |
| Contact update | 联系方式变更 | 影响通知、照会和回访。 |
| Policy loan | 保单贷款 | 赔付时可能涉及欠款扣除。 |
| Surrender | 退保 | 退保后发生事故通常不再承担责任。 |
| Reinstatement | 复效 | 复效后等待期是否重新计算要看条款。 |
| Premium payment | 续期缴费 | 判断保单是否在宽限期、失效或有效状态。 |
| Rider change | 附加险变更 | 影响本次事故可适用的责任。 |

## Maximum Good Faith

保险合同依赖最大诚信原则：

- 公司要清楚说明保险责任、免责、等待期、费用和客户权利。
- 客户要如实告知健康、职业、财务和其他重要事实。
- 理赔时如果发现未如实告知，要判断它是否足以影响承保决定，以及是否与保险事故有严重影响。
